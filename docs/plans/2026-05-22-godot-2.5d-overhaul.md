# Design Document: StoryForge Godot 2.5D Overhaul

## 1. Overview
Transition StoryForge from a web-based (HTML/JS) frontend to a high-fidelity, cinematic Godot Engine client. This overhaul adopts a **2.5D "Living Lorebook"** aesthetic, combining sharp 2D vector character silhouettes with a fully realized 3D tabletop environment.

## 2. Architecture: Brain & Body
*   **The Brain (Python/FastAPI)**: Remains the authoritative server. Manages game rules, D&D 5e mechanics, state persistence (JSON), and Gemini 3.5 Flash AI integration.
*   **The Body (Godot 4.x)**: Replaces the web frontend. Responsible for all visual rendering, particle effects, dynamic lighting, and camera orchestration.

## 3. Visual Aesthetic (2.5D)
*   **The Environment**: 
    *   3D planes with high-resolution parchment textures and normal mapping.
    *   3D props (walls, pillars, furniture) textured with hand-drawn ink styles.
    *   Dynamic 3D lighting with soft ambient glows and hard geometric shadows.
*   **The Characters**:
    *   **Billboard Sprites**: 2D vector/silhouette characters standing vertically in the 3D space, always facing the camera.
    *   **Rim Lighting Shaders**: Glowing outlines to represent Paradox energy.
*   **Cinematic Camera**:
    *   Gimbal-based isometric view.
    *   Dynamic zoom and depth-of-field effects for NPC conversations and critical events.

## 4. Technical Bridge
*   **Communication**:
    *   **HTTP**: For one-off requests (Login, Lobby Join, Character Creation).
    *   **WebSockets**: For real-time state synchronization and AI narration "pushes."
*   **Interpolation**: Godot's `Tween` engine will handle smooth movement animations between grid coordinates.
*   **Packaging**: The final Windows `.exe` will be the Godot binary, which handles launching the Python "Headless DM" as a background process.

## 5. Folder Structure
```text
storyforge/
├── src/                # Python FastAPI Backend
├── godot/              # Godot Project Root
│   ├── scenes/         # Scene files (.tscn)
│   ├── scripts/        # GDScript logic (.gd)
│   ├── assets/         # Shaders, Textures, Meshes
│   └── project.godot   # Godot configuration
└── main.py             # Integrated Launcher
```

## 6. Success Criteria
*   Seamless connection between Godot client and local Python server.
*   Parity with current Lobby/Creation/Exploration features.
*   Drastic improvement in visual feedback (lighting, particles, depth).
*   Zero-config startup for the user (One click runs both).
